# 开发任务规格文档

## 摘要

- 任务按“状态模型 -> 渲染导出 -> 交互 -> UI -> 验证”拆分。
- 每项任务都对应现有分层边界，不跨层乱写。
- 第一版只交付单一独立画笔层，不扩展多图层和高级笔刷引擎。

## 任务清单

### T1：扩展核心状态模型
- 目标：为 `EditorState / HistorySnapshot / SerializableEditorState` 增加画笔字段
- 输出：画笔工具配置、笔触数组、画笔工具状态、主动工具状态
- 验收：
  - 历史快照包含画笔层
  - 草稿恢复包含画笔层

### T2：实现画笔渲染引擎
- 目标：新增 `brush-engine.ts`
- 输出：支持铅笔、画笔、钢笔笔刷、橡皮擦的回放逻辑
- 验收：
  - 支持软硬边
  - 橡皮擦使用擦除合成模式

### T3：接入导出与预览链路
- 目标：在 `createProcessedCanvas` 中绘制独立画笔层
- 输出：底图 + 调整 + 画笔 + 几何变换 + 文字的稳定渲染顺序
- 验收：
  - 导出包含画笔层
  - 文字仍处于最上层

### T4：实现画布交互
- 目标：在 `ImageCanvasEditor` 中新增画笔模式与单笔历史
- 输出：画笔模式、绘制、抬笔提交、整笔撤销/重做
- 验收：
  - 拖拽可连续绘制
  - 单笔为单条历史

### T5：桥接和 UI 控件
- 目标：把画笔状态暴露到 `useImageEditor` 与 `App.vue`
- 输出：工具栏画笔按钮、画笔 Section、参数控件
- 验收：
  - 可以切换工具、颜色、大小、硬度
  - 橡皮擦时颜色控件禁用

### T6：验证
- 目标：补工作流测试并跑构建验证
- 输出：画笔工作流测试、类型检查、构建结果
- 验收：
  - `editor-workflow.test.ts` 覆盖单笔历史和橡皮擦类型
  - `pnpm typecheck` 与 `pnpm build` 通过
