# 技术评审报告

## 摘要

- 方案可行，且不需要引入新框架或额外图层系统。
- 最关键的技术点是“笔触存原图比例点 + 回放到独立画笔层”。
- 已避免直接改底图和每个点一条历史这两种烂方案。
- 风险主要在坐标逆变换和长笔画回放性能，第一版都可控。

## 评审结论

### 通过项
- 分层边界清晰：`editor/core` 持有画笔状态，`editor/vue3` 仅桥接，`apps/web-vue` 仅展示控件
- 历史模型正确：单笔进入 preview，抬笔时 commit
- 导出链路正确：底图调整后绘制画笔，几何变换后再绘制文字

### 主要风险
| 风险 | 影响 | 处理意见 |
|------|------|----------|
| 旋转后输入点映射错位 | 高 | 必须统一使用导出链路同源的裁剪尺寸和逆变换 |
| 长笔画回放开销增长 | 中 | 第一版接受；后续按需做 stroke cache |
| 工具模式冲突 | 中 | 通过 `activeTool` 明确画笔/文字互斥 |

### 不建议做的事
- 不要把画笔直接画到底图像素上
- 不要为了“以后可能多图层”现在就做图层树
- 不要把画笔交互塞进 `apps/web-vue`

## 建议实施顺序
1. 核心状态与历史快照扩展
2. 画笔渲染与导出链路接入
3. 画布交互与坐标逆变换
4. Vue 桥接与 Inspector UI
5. 工作流与构建验证
