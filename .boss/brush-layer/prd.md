## 摘要

> 下游 Agent 请优先阅读本节，需要细节时再查阅完整文档。

- **核心目标**：为当前图片编辑器增加独立画笔图层，支持修图型自由涂抹与擦除，并参与导出合成。
- **目标用户**：需要做局部修饰的轻量修图用户，而不是批注/白板用户。
- **关键功能**：画笔模式、铅笔/画笔/钢笔笔刷/橡皮擦、颜色/大小/硬度、整笔撤销重做、导出合成。
- **技术约束**：必须保持现有 `editor/core -> editor/vue3 -> apps/web-vue` 分层，不引入额外状态库，不破坏裁剪/旋转/文字/滤镜链路。
- **优先级**：MVP 只做单一独立画笔层和基础参数，不做路径钢笔、喷枪、修复画笔、多图层管理。

---

## 1. 概述

### 1.1 背景
当前编辑器已经具备裁剪、旋转、滤镜、文字、导出等能力，但缺少最基础的局部修图能力。用户只能对整张图做全局处理，无法对某一小块区域进行手工修饰。

### 1.2 目标
在不破坏现有编辑体验的前提下，补上“独立画笔图层”能力，让用户能在图上直接进行局部涂抹和擦除，并确保结果与导出一致。

### 1.3 成功指标
- [ ] 用户能在同一工作台中启用画笔并完成连续绘制
- [ ] 用户能切换 4 种基础工具并感知差异
- [ ] 每一笔都能通过撤销/重做完整回退
- [ ] 导出结果包含画笔层，且与预览一致

---

## 2. 用户画像

### 用户画像 1：轻量修图用户
| 属性 | 描述 |
|------|------|
| 角色 | 运营、设计助理、内容编辑、普通修图用户 |
| 特征 | 不需要 Photoshop 级复杂工作流，但需要快速修正局部视觉问题 |
| 需求 | 用简单的笔刷覆盖、强调、修补或擦除局部区域 |
| 痛点 | 只有全局调整，没有局部修图；换工具或软件成本高 |
| 期望 | 打开即用、路径短、撤销可靠、导出一致 |

## 3. 功能需求

### FR-001：进入画笔模式
- **描述**：当用户已有图片且不在裁剪模式下时，系统应允许用户进入画笔模式。
- **优先级**：必须有
- **依赖**：图片已加载
- **验收标准**：
  - [ ] 点击工具栏画笔按钮后进入画笔模式
  - [ ] 画布光标切换为画笔态
  - [ ] 画笔面板显示当前工具和参数

### FR-002：独立画笔图层绘制
- **描述**：系统应将所有笔触记录到独立画笔图层，不直接修改原图像素。
- **优先级**：必须有
- **依赖**：FR-001
- **验收标准**：
  - [ ] 画笔绘制后底图数据不被直接改写
  - [ ] 旋转、裁剪、导出时画笔层与图片保持对齐
  - [ ] 清空/撤销画笔不会影响原图与文字层

### FR-003：基础工具类型
- **描述**：系统应支持铅笔、画笔、钢笔笔刷、橡皮擦四种工具。
- **优先级**：必须有
- **依赖**：FR-001
- **验收标准**：
  - [ ] 用户可在面板中切换四种工具
  - [ ] 各工具至少在硬边、流量或擦除行为上有可感知差异
  - [ ] 橡皮擦只擦除画笔图层内容

### FR-004：基础参数调整
- **描述**：系统应允许用户调整颜色、大小、硬度，并即时影响后续新笔触。
- **优先级**：必须有
- **依赖**：FR-003
- **验收标准**：
  - [ ] 大小支持连续调节
  - [ ] 硬度支持连续调节
  - [ ] 颜色支持颜色选择器和预设色切换
  - [ ] 橡皮擦工具下颜色控件禁用或降级提示

### FR-005：整笔撤销/重做
- **描述**：系统应将一次按下到抬起的连续绘制视为一笔，并参与撤销/重做。
- **优先级**：必须有
- **依赖**：FR-002
- **验收标准**：
  - [ ] 单笔绘制后撤销可完整回退整笔
  - [ ] 重做可恢复整笔
  - [ ] 不因移动中的中间点产生大量历史记录

### FR-006：导出合成
- **描述**：系统应在导出 PNG 时将画笔图层与底图合成。
- **优先级**：必须有
- **依赖**：FR-002、FR-004
- **验收标准**：
  - [ ] 导出图包含画笔效果
  - [ ] 导出图与预览一致
  - [ ] 文字仍然显示在画笔结果之上

## 4. 非功能需求

### NFR-001：兼容性
- **类型**：兼容性
- **描述**：新增画笔能力不能破坏现有裁剪、旋转、文字、滤镜、导出、草稿恢复。
- **指标**：现有核心工作流保持可用

### NFR-002：性能
- **类型**：性能
- **描述**：常规笔触绘制时预览应保持可接受响应。
- **指标**：主流图片尺寸下，单笔连续绘制无明显卡死；构建后预览可正常导出

### NFR-003：可维护性
- **类型**：工程质量
- **描述**：画笔逻辑应留在 `editor/core`，UI 仅负责控制面板与按钮。
- **指标**：不跨层直接读写底层状态；不引入额外状态库

---

## 5. 用户故事

### US-001：局部修图
- **作为** 轻量修图用户
- **我想要** 在图片上直接用画笔涂抹局部区域
- **以便** 快速做出局部修饰而不用切换工具
- **验收标准**：
  - [ ] 进入画笔模式后能直接绘制
  - [ ] 绘制结果立即出现在预览中

### US-002：局部回退
- **作为** 轻量修图用户
- **我想要** 使用橡皮擦和撤销重做修正刚才的笔触
- **以便** 在试错时保持低风险
- **验收标准**：
  - [ ] 橡皮擦只作用于画笔图层
  - [ ] 撤销/重做按整笔生效

### US-003：控制笔触风格
- **作为** 轻量修图用户
- **我想要** 切换工具、颜色、大小、硬度
- **以便** 画出不同风格的局部修饰效果
- **验收标准**：
  - [ ] 参数调整能影响后续笔触
  - [ ] 四种工具在观感上有明显差异

## 6. 范围定义

### 6.1 范围内
- 单一独立画笔图层
- 铅笔 / 画笔 / 钢笔笔刷 / 橡皮擦
- 颜色 / 大小 / 硬度
- 每一笔撤销 / 重做
- PNG 导出合成

### 6.2 范围外
- 路径钢笔工具
- 喷枪、修复画笔、涂抹、仿制图章
- 多图层管理
- 压感输入、纹理散布、高级笔刷参数
- 局部蒙版与智能修图

## 7. 风险与依赖

### 7.1 风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 旋转/裁剪后笔触坐标错位 | 中 | 高 | 笔触统一存储为原图坐标系比例点，再在渲染时回放 |
| 历史记录过碎导致撤销体验差 | 中 | 高 | 使用预览态收集中间点，抬笔时一次提交 |
| 画笔与文字交互互相污染 | 中 | 中 | 画笔模式下清理文字编辑态，保持单一主动工具 |

### 7.2 依赖
| 依赖项 | 类型 | 状态 | 负责人 |
|--------|------|------|--------|
| 现有 `editor/core` 历史快照机制 | 技术 | 就绪 | 当前仓库 |
| 现有 `createProcessedCanvas` 导出链路 | 技术 | 就绪 | 当前仓库 |
| `apps/web-vue` Inspector 面板 | 技术 | 就绪 | 当前仓库 |
