## 摘要

> 下游 Agent 请优先阅读本节，需要细节时再查阅完整文档。

- **核心目标**：为当前图片编辑器增加可编辑、可拖拽、可导出的文字叠加能力，保持现有图片编辑行为不变。
- **目标用户**：需要给单张图片快速加标题、标签或说明文字的普通用户与内容创作者。
- **关键功能**：新增文字、编辑文字内容/字号/颜色、画布拖动定位、导出包含文字、撤销重做支持。
- **技术约束**：不引入后端、不引入新状态库、继续保持 `web-vue / editor-vue / editor-core` 三层边界。
- **优先级**：本期只做单文本 MVP，先把数据结构和渲染链路做对。

---

## 1. 概述

### 1.1 背景
当前编辑器只支持对底图做裁剪、滤镜、旋转、翻转和基础调节，缺少最常见的“给图加字”能力。用户要补文字只能导出后去其他工具继续加工，流程断裂。

### 1.2 目标
- 在当前工作台内直接完成单段文字编辑和摆放。
- 保证文字参与导出，且与图片的裁剪、旋转、翻转结果一致。
- 让文字变更进入现有撤销/重做历史。
- 不破坏已有图片编辑能力和草稿恢复流程。

### 1.3 成功指标
- [x] 用户上传图片后，无需离开当前页面即可完成文字添加并导出。
- [x] 文字编辑与拖拽不会破坏现有裁剪、旋转、滤镜、下载功能。
- [x] 文字新增、修改、移动可被撤销/重做。

## 3. 功能需求

### FR-001：新增文字
- **描述**：当用户已加载图片且当前没有文字时，系统应允许用户一键创建一段默认文字。
- **优先级**：必须有
- **依赖**：已加载图片
- **验收标准**：
  - [ ] AC-1：点击“添加文字”后，画布上出现一段默认文字并进入可编辑状态。
  - [ ] AC-2：没有加载图片时，添加文字入口保持禁用。

### FR-002：编辑文字属性
- **描述**：当用户已创建文字时，系统应允许编辑文字内容、字号和颜色，并实时刷新预览。
- **优先级**：必须有
- **依赖**：FR-001
- **验收标准**：
  - [ ] AC-1：文字输入变化后，画布预览实时更新。
  - [ ] AC-2：字号可通过范围输入调整，颜色可通过颜色输入调整。
  - [ ] AC-3：空字符串视为无效内容，不允许提交为完全空白文字。

### FR-003：拖拽定位
- **描述**：当用户在预览画布上按住文字区域拖动时，系统应更新文字位置，并保持在当前图片可见范围内。
- **优先级**：必须有
- **依赖**：FR-001
- **验收标准**：
  - [ ] AC-1：拖拽文字时预览连续更新。
  - [ ] AC-2：松手后位置被保存，可被撤销。
  - [ ] AC-3：文字不会被拖出原图边界之外。

### FR-004：参与导出
- **描述**：当用户下载 PNG 时，导出图必须包含当前文字叠加结果。
- **优先级**：必须有
- **依赖**：FR-001、FR-002、FR-003
- **验收标准**：
  - [ ] AC-1：导出 PNG 包含当前文字。
  - [ ] AC-2：裁剪、旋转、翻转后的导出结果与预览一致。

### FR-005：历史与草稿兼容
- **描述**：文字相关状态必须进入现有撤销/重做和本地草稿存储。
- **优先级**：必须有
- **依赖**：FR-001、FR-002、FR-003
- **验收标准**：
  - [ ] AC-1：新增文字、修改文字、移动文字均可撤销/重做。
  - [ ] AC-2：保存草稿后刷新页面，恢复草稿仍可得到原文字状态。

## 4. 非功能需求

### NFR-001：性能要求
- **类型**：性能
- **描述**：文字预览刷新应保持编辑器现有交互流畅性。
- **指标**：文字输入和拖拽时不出现明显卡顿；不新增额外网络请求。

### NFR-002：兼容性要求
- **类型**：兼容性
- **描述**：已有无文字草稿、已有编辑历史和原有导出逻辑必须继续可用。
- **指标**：旧草稿可恢复；无文字状态不报错；原有图片编辑功能行为不变。

### NFR-003：可维护性要求
- **类型**：工程
- **描述**：文字能力应继续遵守现有三层边界，UI 不直接操作 Canvas 细节。
- **指标**：核心状态与渲染逻辑位于 `editor/core`，Vue 层只做桥接和控件编排。

## 5. 用户故事

### US-001：快速加标题
- **作为** 需要给海报或商品图加一句文案的用户
- **我想要** 在当前编辑器里直接添加并拖动一段文字
- **以便** 不用再切换到其他设计工具
- **验收标准**：
  - [ ] 上传图片后可以一键添加文字
  - [ ] 拖动后导出结果与当前预览一致

### US-002：反复调整
- **作为** 需要试不同字号和颜色的用户
- **我想要** 实时看到文字修改效果，并能撤销回退
- **以便** 更快找到合适样式
- **验收标准**：
  - [ ] 内容、字号、颜色调整实时生效
  - [ ] 每次提交都能通过撤销回退

## 6. 范围定义

### 6.1 范围内
- 单段文字覆盖层
- 文本内容编辑
- 字号调整
- 文字颜色选择
- 画布内拖拽定位
- 文字参与导出、撤销重做、草稿存储
- 基础选中态和交互提示

### 6.2 范围外
- 多段文字管理
- 字体上传或字体族切换
- 文字旋转、描边、阴影、透明度
- 专门的移动端手势优化
- 富文本或多行排版系统

## 7. 风险与依赖

### 7.1 风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 文字坐标与裁剪/旋转链路不一致 | 中 | 高 | 统一存储在原图坐标系，导出与预览复用同一计算逻辑 |
| 画布拖拽与现有平移手势冲突 | 中 | 中 | 先命中文字再进入拖拽，否则维持原有平移行为 |
| 旧草稿缺少文字字段 | 高 | 中 | 通过可选字段兼容，恢复时提供默认空值 |

### 7.2 依赖
| 依赖项 | 类型 | 状态 | 负责人 |
|--------|------|------|--------|
| 现有 `editor-core` 状态与历史机制 | 技术 | 就绪 | 开发 |
| 现有 Vue 工作台与 Inspector 布局 | 技术 | 就绪 | 开发 |

## 8. 开放问题

- [ ] 本期默认文字是否允许多行输入。当前建议单行，后续版本再扩展。
